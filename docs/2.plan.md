## 1. Executive Summary

This document outlines the plan to integrate the [threads-scraper](https://github.com/vdite/threads-scraper) repository as a polyglot worker within the `ghost-api` ecosystem. The threads-scraper is a powerful Python-based tool using Playwright that provides capabilities not available through the official Threads API, specifically:

- **Recursive deep scraping** of nested replies and sub-threads
- **BFS traversal** to capture entire conversation trees
- **Cookie-based authentication** for expanded data access
- **Smart deduplication** using unique post IDs
- **Dynamic content handling** via Playwright

### Strategic Value

| Capability | Official Threads API | threads-scraper | ghost-api Value |
|------------|---------------------|-----------------|-----------------|
| Nested Replies | Limited | ✅ Full depth | Complete conversation extraction |
| Deep Sub-threads | ❌ | ✅ BFS traversal | Research-grade data collection |
| No Rate Limits | ❌ (CPU quota) | ✅ (IP-based only) | Higher throughput potential |
| Login Bypass | N/A | ✅ Cookie auth | Account-bound data access |
| Lazy-load Handling | N/A | ✅ Playwright | Dynamic content extraction |

### Integration Goals

1. **Wrap threads-scraper as a GhostWorker** implementing the trait interface
2. **Map output to GhostPost/GhostUser** via the threads-adapter
3. **Enable health-based routing** between scraper and official API
4. **Support multi-tenant injection** of cookies/proxies
5. **Maintain scaffolding convention** until business logic implementation

### 2.3 Technical Implementation

#### Core Algorithm (BFS Traversal)

```python
# Simplified representation of the scraping logic
def scrape_thread(url, max_pages=100):
    queue = [url]           # BFS queue
    visited = set()         # Deduplication via post ID
    results = []
    
    while queue and len(visited) < max_pages:
        current_url = queue.pop(0)
        page_content = fetch_with_playwright(current_url)
        
        posts = extract_posts(page_content)
        for post in posts:
            if post.id not in visited:
                visited.add(post.id)
                results.append(post)
                
                # Add sub-thread URLs to queue
                if post.has_replies:
                    queue.extend(post.reply_urls)
    
    return results
```

#### Key Technical Features

1. **Playwright Integration**
   - Headless Chromium browser
   - Handles JavaScript-rendered content
   - Manages lazy-loaded elements
   - Supports cookie injection

2. **GraphQL Response Parsing**
   - Intercepts Meta's internal GraphQL responses
   - Handles `for (;;);` prefix in Ajax responses
   - Extracts structured data from nested JSON

3. **Dynamic Scroll Strategy**
   - Scrolls until no new content
   - Auto-clicks "Load more" buttons
   - Supports both English and German UI

4. **Cookie-Based Authentication**
   - Interactive login flow
   - Session persistence to `threads_cookies.json`
   - Automatic session management

### 3.2 Component Responsibilities

| Component | Responsibility | File Location |
|-----------|----------------|---------------|
| **ghost-core** | Health scoring, routing, fallback logic | `crates/ghost-core/` |
| **ghost-bridge** | PyO3 bindings, Python runtime management | `crates/ghost-bridge/` |
| **threads-scraper** | Raw data extraction from threads.net | `scrapers/py-threads/` (new) |
| **threads-adapter** | Parse PayloadBlob to Ghost types | `crates/ghost-adapters/threads-adapter/` |
| **ghost-schema** | Unified type definitions | `crates/ghost-schema/` |

### 3.3 Data Flow

```
1. Request: ghost.x().get_thread(url, ctx)
      │
      ▼
2. Routing: ghost-core selects py-threads worker (health > 0.8)
      │
      ▼
3. Injection: GhostContext passes cookies/proxy to Python
      │
      ▼
4. Scraping: threads-scraper executes BFS traversal
      │
      ▼
5. Extraction: Raw JSON returned as PayloadBlob
      │
      ▼
6. Adaptation: threads-adapter parses to GhostPost[]
      │
      ▼
7. Return: Unified GhostPost objects to caller
```

### 5.1 GhostWorker Interface

```rust
/// GhostWorker trait implementation for threads-scraper
pub trait GhostWorker: Send + Sync {
    /// Unique identifier for this worker
    fn id(&self) -> &str;
    
    /// Capabilities this worker supports
    fn capabilities(&self) -> Vec<Capability>;
    
    /// Execute a request and return a PayloadBlob
    async fn execute(&self, ctx: &RawContext) -> Result<PayloadBlob, GhostError>;
    
    /// Optional: Health check endpoint
    async fn health_check(&self) -> Result<WorkerHealth, GhostError> {
        Ok(WorkerHealth::default())
    }
}
```
